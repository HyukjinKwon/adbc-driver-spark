# Code of Conduct

## Our pledge

We want participation in the adbc-driver-spark project to be a welcoming and
respectful experience for everyone, regardless of background or identity. We
expect all participants to act professionally and considerately in all project
spaces, including issues, pull requests, discussions, and any other channels.

## Our standard

This project adopts the [Contributor Covenant, version 2.1][cc], a widely used
code of conduct in open source. Please read it. In short: be kind, assume good
faith, give and accept constructive feedback gracefully, and focus on what is
best for the community. Behavior that is disrespectful, harassing, or otherwise
unprofessional is not acceptable.

[cc]: https://www.contributor-covenant.org/version/2/1/code_of_conduct/

## Scope

This Code of Conduct applies within all project spaces and also when an
individual is representing the project in public spaces.

## Reporting

If you experience or witness unacceptable behavior, please report it privately to
the maintainer, Hyukjin Kwon, by opening a
[GitHub Security Advisory](https://github.com/HyukjinKwon/adbc-driver-spark/security/advisories/new)
(which supports private communication) or by contacting the maintainer directly.
All reports will be reviewed and handled confidentially.

## Enforcement

Maintainers are responsible for clarifying and enforcing this Code of Conduct and
may take any action they deem appropriate, including warning the offender or
removing comments, commits, code, and contributions that are not aligned with it.
The enforcement guidelines follow those described in the Contributor Covenant 2.1.
